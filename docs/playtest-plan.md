# 《不许跳！》测试与验收

## D3：核心手感

仅测试一个房间、一把手枪和 PC/手机输入。

记录首次成功离地时间、理解反向冲量的尝试数、主动重试间隔、双端完成同一组三跳的差异，以及玩家能否说出失败原因。

**通过线：**

- 80% 玩家在 60 秒内无需讲解完成第一次目标落地。
- 至少 60% 玩家失败后 3 秒内主动重试。
- 手机三跳用时不超过 PC 的 1.5 倍。
- 玩家把失败归因于角度或力度，而非“游戏随机”。

未通过时只调整输入、冲量、镜头、平台和反馈，不增加武器。[→ game-design-fundamentals：先验证 30 秒循环；Miyamoto：手感优先]

## D9：灰盒流程

- 团队外 3 人静默测试。
- 标记卡住超过 30 秒、连续死亡 5 次、误解机关和漏看落点的位置。
- 熟练开发者目标 8–10 分钟，新玩家 15–25 分钟。
- 任一房间若 2/3 测试者无法说出改进方法，视为设计问题。

[→ level-design：静默测试；Jonathan Blow：挑战必须诚实]

## D14：完整体验

- 测试开场到结算，包含文案、基础美术和音频。
- 记录每区用时、死亡数、重试间隔、武器误用和捷径发现率。
- 追问：最爽的一次落地、最不公平的一次失败、是否愿意挑战更快成绩。
- 若玩家记不住武器差异，删除或重做最弱的一把。

## D18：发布候选

- 至少 5 名外部玩家、2 台手机、1 台 PC。
- 至少 3/5 完成主流程，未完成者应到达区域 3。
- 至少 3/5 愿意立即重试或挑战更好成绩。
- 无阻断 Bug、比例遮挡、字幕超框和明显音量突变。
- 从启动到开始游戏不超过 3 次输入。

## 指标红线

| 指标 | 目标 | 红线 |
|---|---:|---:|
| 首次理解规则 | ≤60 秒 | >120 秒 |
| 失败到恢复 | ≤1.5 秒 | >3 秒 |
| 新玩家主流程 | 15–25 分钟 | >40 分钟 |
| 普通房连续失败中位数 | ≤3 次 | >6 次 |
| 手机/PC 完成时间比 | ≤1.5 | >2.0 |
| 愿意再次挑战 | ≥60% | <40% |

## 每日构建检查

- [ ] 相同输入是否产生相同反冲？
- [ ] 手机安全区与屏幕比例是否无遮挡？
- [ ] 关键音效是否有视觉替代？
- [ ] 重试是否清空上一帧输入和速度？
- [ ] 新内容是否直接服务核心循环？
- [ ] 发布构建是否只引用规定目录内资源？
