# 《不许跳！》D1 制作原型文档

版本：D1 / 手感验证版  
目标读者：程序、美术、文案  
制作周期建议：0.5～1 天  
当前目标：只验证“瞄准—开火—反冲—落地—重试”是否有趣

---

## 1. D1 只做什么

D1 不是游戏成品，也不是 Demo。D1 是一次手感实验。

本次原型只需要回答 4 个问题：

1. 玩家不用跳跃键，只靠开枪反冲移动，是否能在 30 秒内理解？
2. 玩家第一次成功落到平台时，是否有“我会了”的爽感？
3. 玩家失败后，是否愿意马上再试一次？
4. PC 操作是否成立？手机操作只做方向可行性观察，不作为 D1 通过硬门槛。

只要这 4 个问题有 3 个答案是“可以”，这个方向就值得继续。

---

## 2. D1 禁止做什么

为了防止第一天直接膨胀，D1 明确不做：

- 不做剧情演出。
- 不做正式美术。
- 不做多武器。
- 不做排行榜。
- 不做每日挑战。
- 不做存档。
- 不做敌人。
- 不做完整 UI。
- 不做关卡选择。
- 不做数值成长。
- 不做复杂粒子和音效。

D1 的敌人不是技术难题，而是“看起来什么都想顺手做一点”。

---

## 3. D1 最终可玩形态

玩家进入一个按 `960 × 720` 固定逻辑画布设计的单屏房间。D1 尽量一屏内完成，不做相机跟随。

房间里有：

- 一个角色。
- 一把固定参数的反冲手枪。
- 起点平台。
- 3 个目标平台。
- 1 个终点区域。
- 掉落重生区域。
- 简单计时。
- 简单成功提示。

玩家需要从起点出发，通过向相反方向开火产生反冲，依次落到平台，最后到达终点。

推荐 D1 名称：

《入职校准测试》

---

## 4. D1 核心体验

一句话：

玩家不能跳，只能用枪把自己“崩”上去。

30 秒核心循环：

```text
观察平台位置
→ 调整瞄准方向
→ 开火
→ 角色被反冲推出
→ 空中微调
→ 落地成功或掉落失败
→ 立刻重试
```

这个循环里最重要的不是“飞得远”，而是“我知道自己为什么飞错了”。

失败原因必须清楚：

- 角度太低，飞不过去。
- 角度太高，飞行弧线过陡，水平距离不够，错过平台。
- 开火太早，起跳点错了。
- 空中补枪方向错了。
- 落点太窄，落地控制差。

不要让玩家觉得是随机、玄学、物理引擎发疯。

## 5. D1 操作方案

### PC 操作

推荐：

- A / D：左右慢速移动。
- 鼠标移动：瞄准。
- 鼠标左键点击：开火。
- R：立即重生。
- Esc：退出或暂停，可暂时不做。

### 手机操作

推荐：

- 左侧虚拟区域：左右慢速移动。
- 右侧拖拽：显示瞄准方向。拖拽方向表示枪口方向，不是拉弓式反向拖拽。
- 松手：开火，玩家会朝枪口相反方向飞。
- 屏幕上放一个小按钮：重生。

### 关键限制

D1 不做持续开火。

原因：

持续开火会让后坐力变成噪音，玩家控制不住，也很难判断失败原因。D1 先做“单次开火”，让每一次输入都清楚、可学习、可复盘。

## 6. 角色基础参数

这些数值不是最终数值，只是 D1 起点。

| 参数 | 建议值 | 目的 |
|---|---:|---|
| 水平移动速度 | 低 | 让枪反冲成为主移动方式 |
| 地面摩擦 | 中高 | 落地后不乱滑 |
| 空中水平控制 | 很低 | 保留一点修正，但不能抢掉反冲主体 |
| 重力 | 中等偏强 | 让抛物线明显，失败节奏快 |
| 最大下落速度 | 限制 | 避免掉落速度过快导致不好读 |
| 角色碰撞体 | 稍微宽容 | D1 不要卡碰撞 |

程序实现建议：

- 先用简单矩形碰撞。
- 先不要做斜坡。
- 先不要做复杂角色动画状态机。
- 落地判定要稳定，宁可宽容一点。

## 7. D1 武器：校准手枪

D1 只有一把枪。

### 规则

- 每次点击/松手发射 1 发。
- 开火后，角色获得一个与瞄准方向相反的冲量。
- 地面可以开火。
- 空中最多允许补 1 枪。
- 落地后恢复空中补枪次数。
- 不需要弹药。
- 不需要换弹。
- 不需要伤害。

### 推荐参数

| 参数 | 建议 |
|---|---|
| 反冲强度 | 中等 |
| 冷却时间 | 0.25～0.4 秒 |
| 空中补枪次数 | 1 |
| 子弹表现 | 一条短线或小点即可 |
| 后坐力方向 | 严格等于瞄准方向的反方向 |

### 手感原则

玩家应该感觉：

- 向右下方开枪，角色向左上方飞。
- 向正下方开枪，角色向上飞。
- 角度稍微变一点，落点明显变化。

如果角度变化不影响结果，游戏会无聊。  
如果角度变化影响过大，游戏会太难。

## 8. D1 房间设计

D1 只做一个房间，建议横版 2D。

### 房间结构

```text
┌──────────────────────────────┐
│                              │
│                 [终点]       │
│                              │
│            [平台3]           │
│                              │
│      [平台2]                 │
│                              │
│ [起点]       [平台1]         │
│                              │
│~~~~~~~~~~~~ 掉落重生区 ~~~~~~│
└──────────────────────────────┘
```

### 平台 1：理解反冲

目的：

让玩家第一次明白“枪往哪打，人往反方向飞”。

设计：

- 平台 1 比起点略高、略远。
- 玩家只需要一次合理开火就能到。
- 平台要宽。

通过标准：

- 新玩家 60 秒内应能成功到达。

### 平台 2：理解角度

目的：

让玩家知道不同角度会带来不同落点。

设计：

- 平台 2 更高。
- 需要略偏下的射击角度。
- 如果直接朝正下方打，会高度够但距离不够。

通过标准：

- 玩家失败 2～5 次后，能主动调整角度。

### 平台 3：理解空中补枪

目的：

让玩家第一次使用空中补枪修正。

设计：

- 单枪可以接近，但不稳定。
- 使用一次空中补枪会明显更容易。
- 平台宽度中等。

通过标准：

- 玩家能说出“我可以在空中再打一枪修一下”。

### 终点：小高潮

目的：

让玩家用已学内容完成一次短挑战。

设计：

- 终点不需要难。
- 终点前可以放一个稍窄的平台或稍高平台。
- 到达后显示完成时间。

通过标准：

- 玩家完成后愿意再跑一次刷时间。
- 通关后冻结角色，显示成绩；按 R 重新开始本房间。

## 9. D1 失败与重试

D1 必须重试很快。

规则：

- 掉到屏幕底部或掉落区，立即重生到起点平台。
- D1 只有一个重生点：起点平台。不做最近安全点或检查点。
- 重生时间控制在 0.5 秒以内。
- 掉落自动重生不重置计时，只增加事故次数。
- 按 R 可以重新开始本房间，重置计时、事故次数和角色状态。

不要做：

- 长死亡动画。
- 黑屏加载。
- 复杂扣分。
- 从很远的地方重新开始。

失败体验应该是：

“啊，角度错了，再来。”

而不是：

“算了，不想等。”

## 10. D1 最小 UI

只做 4 个 UI 元素：

1. 当前时间。
2. 重生次数。
3. 操作提示。
4. 通关提示。

D1 UI 使用白盒临时文本即可，不做正式 UI 皮肤；程序实现以 NanoVG 临时绘制为准。

### 操作提示文案

PC：

```text
A/D 移动，鼠标瞄准，左键开火。
没有跳跃键。对，真的没有。
```

手机：

```text
左侧移动，右侧拖拽瞄准，松手开火。
没有跳跃键。请相信后坐力。
```

### 通关提示文案

```text
入职校准完成
用时：XX.XX 秒
事故次数：X
评价：基本具备被反冲力伤害的资格
```

## 11. D1 临时美术需求

美术第一天不需要画正式资源。

只需要占位：

- 角色：一个带方向的小方块或胶囊。
- 枪口方向：一条线。
- 平台：纯色矩形。
- 终点：发光矩形或门。
- 掉落区：红色半透明区域。
- 背景：纯色或简单网格。

如果美术想参与，可以做“实验室白盒风格”：

- 浅灰背景。
- 深灰平台。
- 黄色终点。
- 红色危险区。
- 蓝色玩家。

D1 不要画角色立绘，不要做正式场景，不要做 UI 皮肤。

## 12. D1 临时文案需求

文案第一天只需要提供 5 条短句，不要写完整剧情。

用途：

- 开场提示 1 条。
- 第一次失败提示 1 条。
- 到达平台 2 提示 1 条。
- 学会空中补枪提示 1 条。
- 通关评价 1 条。

推荐语气：

轻松、荒诞、像实验室安全手册，但不要黑暗。

示例：

```text
欢迎参加“非跳跃移动安全培训”。
请注意：如果你飞错方向，通常不是枪的问题。
很好，你已经理解了反冲力，也许。
空中补枪不是作弊，是公司批准的求生手段。
校准完成。请前往下一间可能违反常识的房间。
```

## 13. D1 音效需求

可以没有正式音效，但如果有时间，优先做 4 个：

1. 开火声。
2. 落地声。
3. 失败重生声。
4. 通关声。

音效目标：

- 开火要短、干脆、有推动感。
- 落地要让玩家知道“站稳了”。
- 失败不要太刺耳，避免烦躁。

D1 不需要背景音乐。

## 14. 程序实现拆分

推荐按 6 个制作功能模块理解，不要一上来做大架构。实际 Lua 文件结构以 `docs/程序/d1-technical-design.md` 为准。

### 1. 输入模块

负责：

- 读取 PC 瞄准方向。
- 读取 PC 开火。
- 读取左右移动。
- 预留手机拖拽瞄准接口。

输出：

- moveAxis：-1 到 1。
- aimDirection：单位向量。
- firePressed：是否开火。

### 2. 玩家移动模块

负责：

- 水平慢速移动。
- 重力。
- 地面/空中状态。
- 速度更新。
- 最大速度限制。

### 3. 反冲模块

负责：

- 根据 aimDirection 计算 recoilImpulse。
- 给玩家速度添加冲量。
- 控制冷却。
- 控制空中补枪次数。

核心公式：

```text
recoilVelocity = -aimDirection * recoilPower
playerVelocity += recoilVelocity
```

### 4. 碰撞与落地模块

负责：

- 平台碰撞。
- 是否在地面。
- 落地后恢复空中补枪。
- 防止角色穿透平台。

### 5. 重生模块

负责：

- 掉落检测。
- R 键重生。
- 重置位置、速度、空中补枪次数。
- 记录重生次数。

### 6. 计时与完成模块

负责：

- 开局计时。
- 到达终点停止计时。
- 显示完成文本。

## 15. 推荐开发顺序

严格按这个顺序做：

1. 角色能在空房间里受重力下落。
2. 角色能站在平台上。
3. A/D 能慢速移动。
4. 鼠标或拖拽能产生瞄准方向。
5. 点击开火能产生反冲。
6. 加入空中最多补 1 枪。
7. 加入掉落重生。
8. 搭 3 个平台和终点。
9. 加计时和重生次数。
10. 找人试玩 5 分钟。

做到第 5 步就已经可以开始判断方向。

## 16. D1 验收标准

D1 完成不以“功能齐全”为标准，而以可测试为标准。

### 必须达成

- 玩家可以从起点到终点。
- 玩家不能使用普通跳跃。
- 开火反冲方向清楚。
- 失败后 0.5 秒内能重试。
- PC 可玩。

### 尽量达成

- 手机拖拽瞄准可玩，且玩家能理解拖拽方向就是枪口方向。
- 有计时。
- 有重生次数。
- 有简单通关提示。

### 不要求达成

- 画面好看。
- 剧情完整。
- 数值最终平衡。
- 发布级性能。
- 所有平台适配完成。

## 18. D1 通过 / 失败判断

### 通过

满足以下任意 3 条，就算 D1 通过：

- 试玩者 30 秒内理解“开枪反冲移动”。
- 试玩者第一次失败后马上重试。
- 试玩者成功落到平台时有明显正反馈。
- 试玩者愿意再跑一次刷时间。
- 如果测试手机，只记录方向可行性和玩家反馈；手机不计入 D1 硬性通过标准。

### 需要调整

出现以下情况，先调数值，不要推翻方向：

- 玩家总是飞太远：降低反冲或提高重力。
- 玩家总是飞不动：提高反冲或降低重力。
- 玩家觉得控制玄学：减少空中控制，强化瞄准线和轨迹反馈。
- 玩家落地很烦：加宽平台，优化碰撞，增加落地缓冲。

### 暂停方向

如果出现以下情况，才考虑大改：

- 多数玩家理解了规则但觉得无聊。
- 多数玩家不愿意第二次尝试。
- 反冲移动在 PC 和手机上都很别扭。
- 成功和失败都没有情绪波动。
