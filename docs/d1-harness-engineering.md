## 1. 验收结论规则

D1 工程验收只给 3 种结论：

| 结论 | 含义 |
|---|---|
| PASS | 可以交给 1～3 个外部试玩者测试 |
| CONDITIONAL PASS | 可以试玩，但必须记录已知问题 |
| FAIL | 不建议试玩，先修阻断问题 |

### PASS 条件

必须同时满足：

- PC 端完整可玩。
- 玩家可以从起点到终点。
- 没有普通跳跃键。
- 反冲方向正确且稳定。
- 掉落能快速回到起点。
- 通关能冻结并显示成绩。
- 没有阻断型崩溃或卡死。

### CONDITIONAL PASS 条件

允许存在：

- 手机输入暂未完成。
- 音效、粒子、正式 UI 暂缺。
- 数值还不舒服，但核心循环能跑通。
- 个别非阻断显示问题。

### FAIL 条件

出现任意一条即 FAIL：

- 无法从起点到终点。
- 开火方向和反冲方向不稳定或相反。
- 掉落后无法重试。
- 通关后计时或状态混乱。
- 玩家可以通过普通跳跃完成。
- 经常穿平台、卡死、飞出后无法恢复。

## 2. 验收前准备

验收前确认：

- 当前测试对象是 D1 原型，不是完整游戏。
- 使用 PC 作为硬验收平台。
- 手机只做可行性观察，不作为 D1 硬门槛。
- 使用 `960 × 720` 固定逻辑画布。
- D1 使用白盒图形即可，不看正式美术质量。

建议测试环境：

```text
平台：PC
输入：键盘 + 鼠标
房间：D1 单房间
武器：校准手枪
目标：从起点到终点
```

| 编号 | 检查项 | 期望结果 | 结果 |
|---|---|---|---|
| S-001 | 启动 D1 场景 | 无报错、无黑屏 |  |
| S-002 | 玩家出生 | 玩家位于起点平台 |  |
| S-003 | 画面显示 | 可看到玩家、平台、终点、HUD |  |
| S-004 | A/D 输入 | 玩家能左右慢速移动 |  |

## 4. 核心手感验收

### H-001：反冲方向正确

步骤：

1. 玩家站在起点平台。
2. 鼠标放在玩家正下方。
3. 左键开火。

期望：

```text
枪口方向：向下
玩家运动：向上
```

失败表现：

- 玩家向下飞。
- 玩家没有明显位移。
- 玩家方向随机。

---

### H-002：斜向反冲正确

步骤：

1. 鼠标放在玩家右下方。
2. 左键开火。

期望：

```text
枪口方向：右下
玩家运动：左上
```

再测一次左下：

```text
枪口方向：左下
玩家运动：右上
```

验收重点：

- 同样输入得到同样结果。
- 没有随机散布。
- 没有持续开火造成失控。

---

### H-003：空中最多补 1 枪

步骤：

1. 地面开火起飞。
2. 空中再开火一次。
3. 空中尝试第三次开火。

期望：

- 第一次地面开火成功。
- 第二次空中补枪成功。
- 第三次空中开火失败或无效。
- 落地后恢复补枪次数。

失败表现：

- 空中可以无限开火。
- 空中完全不能补枪。
- 落地后没有恢复补枪。

---

### H-004：没有普通跳跃键

步骤：

1. 尝试按 Space / W / 上方向键。
2. 不开火，只尝试跳跃。

期望：

- 玩家不能普通跳跃。
- 垂直跨越只能依靠反冲。

失败表现：

- Space 可以跳。
- W 可以跳。
- 上方向键可以跳。

---

## 5. 移动与碰撞验收

### C-001：平台落地稳定

步骤：

1. 从起点开火到平台 1。
2. 多次尝试落在平台中央和边缘。

期望：

- 中央落地稳定。
- 边缘有轻微宽容。
- 落地后速度垂直分量归零。
- 落地后恢复空中补枪次数。

失败表现：

- 明显落到平台上仍掉下去。
- 从平台侧面撞上也被吸到平台顶。
- 从平台下方向上穿过时被判定落地。

---

### C-002：高速下落不穿平台

步骤：

1. 让玩家从较高位置下落。
2. 尝试以较快速度落到平台。

期望：

- 不应频繁穿过平台。
- 如果偶发穿透，需要记录具体位置和速度情况。

技术判断：

- 应使用 `previousPosition` 做脚底穿越判定。
- 不应只依赖当前帧 AABB overlap。

---

### C-003：掉落区重生

步骤：

1. 故意错过平台并掉落。
2. 观察重生行为。

期望：

- 掉落到 `fallY` 以下后自动回到起点。
- 掉落不重置计时。
- 掉落增加事故次数。
- 重生后速度清零。
- 重生后空中补枪次数清零。

失败表现：

- 掉出屏幕后无法恢复。
- 重生后仍保留旧速度。
- 掉落后计时被错误重置。
- 事故次数不变。

---

## 6. 计时、重生、通关验收

### R-001：R 键重新开始

步骤：

1. 游玩 5 秒。
2. 按 R。

期望：

- 玩家回到起点。
- 速度清零。
- 计时归零。
- 事故次数归零。
- 可以重新开始挑战。

---

### R-002：掉落重生不重置计时

步骤：

1. 游玩 5 秒。
2. 故意掉落。
3. 观察计时和事故次数。

期望：

- 计时继续。
- 事故次数 +1。
- 玩家回到起点。

---

### F-001：通关冻结

步骤：

1. 到达终点区域。
2. 观察角色、计时、HUD。

期望：

- 角色冻结或停止接受玩法输入。
- 计时停止。
- 显示完成时间。
- 显示事故次数。
- 左键开火无效。
- R 可以重新开始。

失败表现：

- 通关后仍继续计时。
- 通关后还能继续开火影响状态。
- 重复触发通关事件导致成绩闪烁。
- R 无法重新开始。

---

## 7. 输入与瞄准验收

### I-001：鼠标坐标转换正确

步骤：

1. 鼠标移动到玩家右侧。
2. 鼠标移动到玩家左侧。
3. 鼠标移动到玩家下方。
4. 观察瞄准线。

期望：

- 瞄准线从玩家中心指向鼠标位置。
- 画布缩放或窗口尺寸变化后，瞄准线仍准确。
- 鼠标位置不会因为 `960 × 720` 适配而偏移。

失败表现：

- 瞄准线和鼠标明显错位。
- 窗口缩放后瞄准方向错误。
- Y 轴方向反了。

---

### I-002：手机输入观察项，可选

如果 D1 已接入手机输入，做此项。否则记录为“未测，不影响 D1 硬验收”。

步骤：

1. 左侧拖动控制水平移动。
2. 右侧拖拽瞄准。
3. 松手开火。

期望：

- 右侧拖拽方向表示枪口方向。
- 松手时开火。
- 拖拽太短时不触发开火。
- 玩家理解自己会朝枪口相反方向飞。

## 8. UI / 白盒显示验收

D1 UI 不验收美观，只验收信息是否够用。

| 编号 | 检查项 | 期望结果 | 结果 |
|---|---|---|---|
| U-001 | 计时显示 | 能看到当前用时 |  |
| U-002 | 事故次数 | 掉落后数字增加 |  |
| U-003 | 操作提示 | 能看到基础操作说明 |  |
| U-004 | 通关提示 | 通关后显示成绩 |  |
| U-005 | 瞄准线 | 能明确看到枪口方向 |  |
| U-006 | 反冲方向箭头，可选 | 如果玩家误解方向，则建议开启 |  |

允许：

- 使用 NanoVG 临时文本。
- 纯色矩形平台。
- 纯色矩形玩家。

不要求：

- 正式 UI 皮肤。
- 动画。
- 粒子。
- 完整音效。

## 9. D1 试玩前工程检查表

交给外部试玩者前，程序自查：

- [ ] PC 能从启动到通关完整跑一遍。
- [ ] 没有普通跳跃键。
- [ ] 反冲方向为 `-aimDir`。
- [ ] 空中最多补 1 枪。
- [ ] 落地恢复补枪次数。
- [ ] 掉落自动回到起点。
- [ ] 掉落不重置计时。
- [ ] 掉落增加事故次数。
- [ ] R 重开会清空计时和事故次数。
- [ ] 通关后冻结并显示成绩。
- [ ] 瞄准线不会因窗口缩放偏移。
- [ ] 平台落地不频繁穿透。
- [ ] 没有明显崩溃、卡死、不可恢复状态。

全部通过后，才进入 1～3 人小范围试玩。

## 10. 试玩记录 Harness

外部试玩时，只记录最少信息。

| 试玩者 | 是否 30 秒内理解规则 | 第一次到平台1用时 | 是否主动重试 | 最大卡点 | 是否愿意再跑一次 | 一句话反馈 |
|---|---|---:|---|---|---|---|
| A | 是/否 |  | 是/否 |  | 是/否 |  |
| B | 是/否 |  | 是/否 |  | 是/否 |  |
| C | 是/否 |  | 是/否 |  | 是/否 |  |

观察原则：

- 不讲解玩法。
- 如果玩家问“怎么跳？”，只说：“没有跳跃键。”
- 不纠正玩家瞄准方式，先观察他能不能从反馈中学会。
- 重点看行为，不只听口头评价。

## 11. 问题分级

### P0：阻断问题

必须马上修。

- 无法启动。
- 无法通关。
- 掉落后无法恢复。
- 反冲方向错误。
- 普通跳跃键存在。
- 通关后状态混乱。

### P1：严重体验问题

试玩前尽量修。

- 平台频繁穿透。
- 瞄准线明显偏移。
- 掉落重生超过 0.5 秒。
- 空中补枪次数错误。
- 玩家经常不知道为什么失败。

### P2：可带入试玩的问题

可以记录，D1 后修。

- UI 不美观。
- 暂无音效。
- 暂无手机输入。
- 个别平台位置太难。
- 文案还不完整。

## 13. 最终验收口径

D1 不是验收“完成度”，而是验收“能不能测手感”。

只要玩家能在 PC 上完成这条闭环：

```text
看平台 → 瞄准 → 开火 → 被反冲弹起 → 落地 / 掉落 → 快速重试 → 到达终点 → 看成绩 → R 再来
```

并且没有阻断问题，就可以进入小范围试玩。
